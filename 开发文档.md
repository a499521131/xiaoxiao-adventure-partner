# 小小冒险伙伴开发文档

## 项目定位

这是一个个人开发者可落地的儿童 AI 伙伴 Web Demo。

当前不是做完整教育系统，不是做 App，不是做硬件，也不是做数字身份平台。第一阶段只做一个能演示的小闭环：

孩子创建伙伴，完成一个知识小挑战，伙伴发生故事变化，系统记录一页回忆。

## 当前产品定义

孩子画出一个属于自己的伙伴，系统把它变成统一美术风格的 2D 卡通形象。孩子通过知识小挑战、互动聊天、游戏反馈和回忆记录，让这个伙伴逐步发生变化。

## 当前开发范围

只保留 6 个系统：

1. 形象设计系统
2. 知识系统
3. 成长系统
4. 游戏系统
5. 记忆系统
6. 硬件桥接系统

## 第一版页面

只做 5 个页面：

1. 创建伙伴
2. 伙伴主页
3. 知识小挑战
4. 互动聊天
5. 回忆故事本

## 当前不做

- 不做原生 App
- 不做真实 3D 自动生成
- 不做复杂骨骼绑定
- 不做完整语音包
- 不做实体硬件
- 不做账号系统
- 不做支付系统
- 不做家长后台
- 不做公开作品集
- 不做数字身份绑定
- 不做真实儿童数据采集
- 不做学校替代方案

## 项目分类

### A 类：当前必须做

- 形象设计系统
- 知识系统
- 游戏系统
- 记忆系统
- 成长系统
- 硬件桥接系统

### B 类：暂缓做

- 3D 形象生成与骨骼绑定
- 20 个交互动作
- 语音包
- 真实儿童试点
- 家长后台

### C 类：归档为背景

- 替代学校的完整教育体系
- 90 天试点计划
- 本地生活网络
- 导师网络
- 数字身份和长期资产
- 商业生态和融资叙事

### D 类：停止维护

- 过大的项目群拆分
- 过完整的教育系统架构
- 过早的试点和线下计划

## 任务派发清单

### T01 产品原型与页面流程

适合：产品经理、UX 设计师。

目标：把 5 个页面流程画清楚，让开发能直接照着做。

交付物：

- 页面流程图
- 低保真线框图
- 页面字段清单

验收标准：

- 不超过 5 个核心页面
- 不出现登录、支付、家长后台
- 能完成“创建伙伴 - 做挑战 - 看到变化 - 保存回忆”的闭环

### T02 伙伴美术风格规范

适合：插画师、视觉设计师。

目标：定义统一的伙伴美术风格，避免生成结果混乱。

交付物：

- 美术风格说明
- 3 到 5 张参考图
- 1 张标准角色样例
- 1 张禁止风格示例表

验收标准：

- 多个角色看起来属于同一个产品世界
- 不依赖具体 IP
- 能指导 AI 绘图提示词

### T03 2D 伙伴生成提示词

适合：AI 绘图提示词设计师。

目标：根据孩子草稿和文字设定，生成统一风格的 2D 伙伴图。

交付物：

- 主提示词
- 负面提示词
- 角色一致性提示词
- 3 到 5 个测试案例

验收标准：

- 风格一致
- 能保留草稿主要特征
- 不生成恐怖、成人化、过度复杂形象

### T04 Web Demo 前端

适合：前端工程师。

目标：实现一个能本地运行的 Web Demo。

交付物：

- 前端代码
- README 运行说明
- 示例数据

验收标准：

- 新用户 3 分钟内能跑完整个 Demo
- 刷新页面后基础数据不丢
- 不依赖付费服务才能打开页面

### T05 本地数据结构

适合：前端或全栈工程师。

目标：定义第一版本地数据结构，支撑伙伴、挑战、记忆和成长事件。

交付物：

- JSON Schema 或 TypeScript 类型定义
- 示例数据
- 字段说明

验收标准：

- 能支撑 5 个页面
- 不包含真实姓名、身份证、地址、学校等敏感字段
- 后续能迁移到数据库

### T06 知识小挑战模板

适合：教育内容设计师。

目标：设计第一批知识小挑战，让家长看出学习价值，孩子又不觉得像做题。

交付物：

- 20 个挑战模板
- 模板字段说明
- 3 个完整示例

验收标准：

- 每个挑战 5 到 15 分钟可完成
- 有明确知识点
- 能触发伙伴技能或故事变化

### T07 游戏反馈与伙伴变化

适合：游戏策划。

目标：设计轻量游戏反馈，让孩子完成挑战后有期待感。

交付物：

- 伙伴变化规则
- 技能命名规则
- 30 个技能或道具名称
- 10 个故事变化模板

验收标准：

- 奖励和知识挑战有关
- 不显示数值
- 不制造攀比

### T08 外部 AI 桥接提示词

适合：Prompt 工程师。

目标：让系统能借用豆包、DeepSeek 等外部 AI 工具。

交付物：

- 知识解释提示词
- 挑战生成提示词
- 伙伴故事改写提示词
- 输出 JSON 格式示例

验收标准：

- 可复制到豆包或 DeepSeek 使用
- 输出内容适合儿童
- 不泄露孩子隐私

### T09 记忆与成长事件规则

适合：产品经理、系统策划。

目标：定义哪些互动会被记录，以及如何影响内部成长状态。

交付物：

- 事件类型表
- 内部状态变化表
- 外显反馈模板

验收标准：

- 不显示任何数值
- 不生成孩子评价报告
- 只用于伙伴反馈和记忆连续性

### T10 基础安全与隐私文案

适合：产品经理、法律或隐私顾问。

目标：为 Demo 提供最低限度的安全和隐私边界说明。

交付物：

- 隐私提示文案
- 上传图片提示文案
- 外部 AI 桥接提示文案
- 儿童使用提示文案

验收标准：

- 简短易懂
- 不吓人
- 不承诺做不到的安全能力

### T11 演示样例与测试脚本

适合：测试、运营、产品助理。

目标：准备一套演示材料，让别人能快速理解产品。

交付物：

- 3 套演示数据
- 1 条完整演示流程脚本
- 10 条测试检查项

验收标准：

- 可用于录屏演示
- 可用于开发测试
- 不包含真实儿童隐私

### T12 项目展示页文案

适合：文案、产品营销。

目标：写一页对外展示文案，说明这个 Demo 是什么。

交付物：

- 一页展示文案
- 3 个标题方案
- 5 条核心卖点

验收标准：

- 不夸大
- 不承诺替代学校
- 不使用焦虑营销
- 普通人 1 分钟能看懂

### T13 法律与知识产权保护

适合：法律从业者、知识产权顾问、熟悉创意项目协作协议的人。

目标：为项目早期协作建立基本的知识产权和贡献归属边界，保护项目概念、文档、角色设定、美术风格、提示词、代码和协作成果。

交付物：

- 早期协作者协议草案
- 知识产权归属说明
- 保密与公开边界说明
- 投稿/交付成果授权条款
- 对外发布前的风险提示

验收标准：

- 能保护项目方向和核心概念
- 能明确协作者交付物的使用权
- 不把协作者权益写死或压榨
- 语言普通人能看懂
- 适合早期无公司、无资金、个人开发者阶段使用

## 给协作者的一句话

这是一个儿童 AI 伙伴 Demo：孩子创建自己的伙伴，通过知识挑战和互动，让伙伴产生故事变化。当前只做 Web 原型，不做完整教育平台。
