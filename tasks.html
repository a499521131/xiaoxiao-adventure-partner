任务协作窗口

给“小小冒险伙伴”用的轻量协作页。它把任务说成人话，方便发给独立开发者、设计师、内容协作者；也能接收完成结果，按清晰标准打分，算出每个人在项目里的贡献权重。

查看项目介绍 查看 GitHub 仓库
当前版本：本地静态页
数据保存：浏览器本地
适合阶段：零资金招募协作

选择要派出去的任务

先选角色或优先级，再点任务卡。右侧会生成一段适合直接发给协作者的话。

任务说明

这里故意不用太专业的话，重点是让对方知道“帮我做什么、交什么、怎么算做好”。

接收完成的任务

把协作者交回来的内容登记在这里。评分不是评价人，而是为了客观记录这个任务对项目的帮助。

验收勾选

打分

完成度交付物是否完整，是否覆盖任务要求。
可用度能不能直接进入项目，是否少改就能用。
理解度是否理解“小小冒险伙伴”的方向，没有做偏。
主动性是否主动补充了有价值的想法或问题。
建议：分数只是项目管理工具，不要把它包装成“人的价值”。贡献权重只用于这个项目内部协作记录。

贡献权重

权重 = 任务基础权重 × 综合得分。系统会按所有记录自动计算每个人占比。
