# 任务派发规格

本文档用于把项目拆成可以派发给不同协作者的小任务包。

当前目标不是做完整系统，而是做一个个人开发者可控的 Web Demo：

孩子创建一个 2D 伙伴，完成一个知识小挑战，伙伴发生变化，系统记录一页回忆故事。

## 派发原则

给协作者派任务时，只说当前任务，不讲完整宏大愿景。

每个任务必须满足：

- 有明确边界
- 有可交付文件
- 有验收标准
- 不依赖真实儿童数据
- 不要求自研硬件
- 不要求完整大模型服务
- 不引入复杂账号、支付、家长后台

## T01 产品原型与页面流程

### 目标

把第一版 Web Demo 的 5 个页面流程画清楚，让开发能直接照着做。

### 具体要求

页面只包含：

1. 创建伙伴
2. 伙伴主页
3. 知识小挑战
4. 互动聊天
5. 回忆故事本

每个页面要写清：

- 页面目的
- 用户输入
- 系统输出
- 主要按钮
- 空状态
- 成功状态
- 错误状态
- 下一步跳转

### 交付物

- 一份页面流程图
- 一份低保真线框图
- 一份页面字段清单

### 验收标准

- 不超过 5 个核心页面
- 不出现登录、支付、家长后台
- 用户能完成“创建伙伴 - 做挑战 - 看到变化 - 保存回忆”的闭环

## T02 伙伴美术风格规范

### 目标

定义统一的伙伴美术风格，避免每个孩子生成出来的角色风格混乱。

### 具体要求

风格要适合儿童，但不要幼稚廉价。

需要定义：

- 角色比例
- 线条风格
- 色彩范围
- 眼睛和表情规则
- 材质质感
- 服饰或装饰复杂度
- 背景是否保留
- 禁止出现的风格

### 交付物

- 美术风格说明
- 3 到 5 张参考图
- 1 张标准角色样例
- 1 张禁止风格示例表

### 验收标准

- 能指导 AI 绘图提示词
- 不依赖具体 IP
- 不像普通头像生成器
- 多个角色看起来属于同一个产品世界

## T03 2D 伙伴生成提示词

### 目标

根据孩子上传的草稿和文字设定，生成统一风格的 2D 伙伴图。

### 具体要求

输入：

- 草稿图
- 伙伴名字
- 性格
- 能力
- 喜欢的颜色

输出：

- 统一风格 2D 伙伴正面图
- 透明或纯色背景
- 适合放在网页伙伴主页中

### 交付物

- 主提示词
- 负面提示词
- 角色一致性提示词
- 3 到 5 个测试案例

### 验收标准

- 同一批生成图风格一致
- 能保留孩子草稿的主要特征
- 不生成恐怖、成人化、过度复杂形象
- 适合后续扩展表情和动作

## T04 Web Demo 前端

### 目标

实现一个能本地运行的 Web Demo。

### 具体要求

必须包含：

- 创建伙伴页面
- 伙伴主页
- 知识小挑战页面
- 互动聊天页面
- 回忆故事本页面

技术要求：

- 可本地运行
- 使用本地存储
- 不需要后端账号
- 适配手机和桌面宽度
- UI 清楚，不做复杂动效

### 交付物

- 前端代码
- README 运行说明
- 示例数据

### 验收标准

- 新用户 3 分钟内能跑完整个 Demo
- 刷新页面后基础数据不丢
- 没有明显布局错乱
- 不依赖付费服务才能打开页面

## T05 本地数据结构

### 目标

定义第一版本地数据如何保存，支撑伙伴、挑战、记忆和成长事件。

### 具体要求

至少包含：

- child_profile
- companion_profile
- challenge_records
- memory_entries
- growth_events
- bridge_prompts

不要保存真实姓名、身份证、地址、学校等敏感信息。

### 交付物

- JSON Schema 或 TypeScript 类型定义
- 示例数据
- 字段说明

### 验收标准

- 能支撑 5 个页面
- 能保存多次挑战和回忆
- 不包含不必要敏感字段
- 后续能迁移到数据库

## T06 知识小挑战模板

### 目标

设计第一批知识小挑战，让家长能看出学习价值，孩子又不觉得像做题。

### 具体要求

先做 20 个模板。

每个模板包含：

- 适合年龄
- 知识主题
- 一句话解释
- 一个小问题
- 一个小行动
- 完成后的伙伴变化

主题建议：

- 昆虫
- 声音
- 光影
- 植物
- 形状
- 数字
- 天气
- 情绪
- 记忆
- 空间

### 交付物

- 20 个挑战模板
- 模板字段说明
- 3 个完整示例

### 验收标准

- 每个挑战 5 到 15 分钟可完成
- 不像刷题
- 有明确知识点
- 能触发伙伴技能或故事变化

## T07 游戏反馈与伙伴变化

### 目标

设计轻量游戏反馈，让孩子完成挑战后有期待感。

### 具体要求

第一版只做：

- 新表情
- 新道具
- 新技能名称
- 新故事片段
- 伙伴房间小变化

不要做：

- 排名
- 战斗力展示
- 抽卡付费
- 孩子之间对比
- 复杂对战

### 交付物

- 伙伴变化规则
- 技能命名规则
- 30 个技能或道具名称
- 10 个故事变化模板

### 验收标准

- 奖励和知识挑战有关
- 不依赖数值显示
- 孩子能理解“伙伴发生了变化”
- 不制造攀比

## T08 外部 AI 桥接提示词

### 目标

让系统能借用豆包、DeepSeek 等外部 AI 工具，而不自建模型服务。

### 具体要求

设计三类可复制提示词：

1. 知识解释提示词
2. 挑战生成提示词
3. 伙伴故事改写提示词

提示词必须要求外部 AI 输出结构化结果，方便粘贴回系统。

### 交付物

- 3 类桥接提示词
- 输出 JSON 格式示例
- 粘贴回系统后的解析字段说明

### 验收标准

- 用户复制到豆包或 DeepSeek 后能得到可用结果
- 输出内容适合儿童
- 不要求外部 AI 知道完整项目背景
- 不泄露孩子隐私

## T09 记忆与成长事件规则

### 目标

定义哪些互动会被记录，以及如何影响内部成长状态。

### 具体要求

记录事件类型：

- create_companion
- learn_topic
- complete_challenge
- chat_emotion
- refuse_suggestion
- return_to_memory
- idle_loop

每类事件要定义：

- 触发条件
- 记录字段
- 内部状态变化
- 外显反馈文案

### 交付物

- 事件类型表
- 内部状态变化表
- 外显反馈模板

### 验收标准

- 不显示任何数值
- 不生成孩子评价报告
- 只用于伙伴反馈和记忆连续性
- 能被前端工程师实现

## T10 基础安全与隐私文案

### 目标

为 Demo 提供最低限度的安全和隐私边界说明。

### 具体要求

写清楚：

- 当前 Demo 不收集真实儿童身份
- 不要输入真实姓名、学校、地址、电话
- 上传图片仅用于演示
- 本地存储数据由用户自己保管
- 外部 AI 桥接时不要粘贴隐私内容

### 交付物

- 隐私提示文案
- 上传图片提示文案
- 外部 AI 桥接提示文案
- 儿童使用提示文案

### 验收标准

- 简短易懂
- 不吓人
- 不承诺做不到的安全能力
- 能直接放进页面

## T11 演示样例与测试脚本

### 目标

准备一套演示材料，让别人能快速理解产品。

### 具体要求

准备 3 个样例孩子设定，但不要使用真实儿童信息。

每个样例包含：

- 草稿描述
- 伙伴设定
- 一个知识主题
- 一个挑战
- 一条回忆故事

### 交付物

- 3 套演示数据
- 1 条完整演示流程脚本
- 10 条测试检查项

### 验收标准

- 可以用于录屏演示
- 可以用于开发测试
- 不包含真实儿童隐私

## T12 项目展示页文案

### 目标

写一页对外展示文案，说明这个 Demo 是什么。

### 具体要求

文案面向普通家长和潜在协作者。

必须讲清：

- 它不是网课
- 它不是题库
- 它不是普通 AI 聊天
- 它是孩子自己创造伙伴，并通过知识挑战让伙伴变化的 Demo

### 交付物

- 一页展示文案
- 3 个标题方案
- 5 条核心卖点

### 验收标准

- 不夸大
- 不承诺替代学校
- 不使用焦虑营销
- 普通人 1 分钟能看懂

## T13 法律与知识产权保护

### 目标

为项目早期协作建立基本的知识产权和贡献归属边界，保护项目概念、文档、角色设定、美术风格、提示词、代码和协作成果。

### 具体要求

需要考虑：

- 项目概念和命名如何保护
- 对外沟通稿、开发文档、任务文档如何声明权利
- 协作者交付的设计、代码、提示词、内容模板归谁使用
- 协作者是否保留署名权
- 未采用稿件如何处理
- 公开分享与保密内容的边界
- 后续成立公司或开源时如何过渡

### 交付物

- 早期协作者协议草案
- 知识产权归属说明
- 保密与公开边界说明
- 投稿/交付成果授权条款
- 对外发布前的风险提示

### 验收标准

- 能保护项目方向和核心概念
- 能明确协作者交付物的使用权
- 不把协作者权益写死或压榨
- 语言普通人能看懂
- 适合早期无公司、无资金、个人开发者阶段使用

## 建议派发顺序

第一批先派：

1. T01 产品原型与页面流程
2. T02 伙伴美术风格规范
3. T05 本地数据结构
4. T06 知识小挑战模板

第二批再派：

1. T03 2D 伙伴生成提示词
2. T04 Web Demo 前端
3. T07 游戏反馈与伙伴变化
4. T08 外部 AI 桥接提示词

第三批最后派：

1. T09 记忆与成长事件规则
2. T10 基础安全与隐私文案
3. T11 演示样例与测试脚本
4. T12 项目展示页文案

并行优先寻找：

1. T13 法律与知识产权保护

## 当前不要派发的任务

暂时不要派：

- App 开发
- 3D 自动生成
- 骨骼绑定
- 语音包
- 硬件开发
- 支付系统
- 家长后台
- 真实儿童试点
- 数字身份
- 区块链或资产确权
- 线下活动

原因：这些都会把第一版复杂度拉爆。

## 给协作者的一句话说明

这是一个儿童 AI 伙伴 Demo：孩子创建自己的伙伴，通过知识挑战和互动，让伙伴产生故事变化。当前只做 Web 原型，不做完整教育平台。
