# 落地系统、硬件与模型栈

这份文档回答三个落地问题：

1. 这个教育系统到底包含哪几个产品系统
2. 它通过什么硬件进入孩子生活
3. 它使用什么大模型和模型路由

## 一句话产品形态

完整人格教育不是一个网课平台，而是一个面向孩子日常生活的成长操作系统。

它以“AI 成长伙伴”为入口，通过手机、平板、家庭终端、可穿戴设备和项目工具包进入孩子生活，再把孩子带向真实世界任务、同伴协作和长期作品集。

## 1. 产品系统组成

### 1. AI 成长伙伴系统

这是孩子每天接触最多的系统。

它的功能：

- 日常对话
- 情绪陪伴
- 兴趣发现
- 问题生成
- 项目拆解
- 知识解释
- 复盘追问
- 成长报告

它的形象不是老师，而是长期陪孩子探索世界的伙伴。

### 2. 成长档案系统

这是整个系统的个人数据库。

记录内容：

- 孩子的兴趣变化
- 问题清单
- 项目经历
- 作品证据
- 同伴合作记录
- 情绪和生命力状态
- 能力变化
- 成人观察
- AI 阶段总结

原则：

- 孩子拥有自己的成长档案
- 家长有监护和支持视图
- 系统只记录成长需要的信息
- 敏感内容分级保存
- 不做无意义监控

### 3. 项目任务引擎

这是替代课程表的系统。

它不按“语文、数学、英语”组织，而按真实项目组织。

项目来源：

- 孩子的兴趣
- 家庭生活
- 社区问题
- 自然观察
- 创作任务
- 技术制作
- 公益服务
- 真实用户需求

每个项目包含：

- 问题
- 目标
- 任务拆解
- 知识调用
- 最小可交付成果
- 外部反馈对象
- 作品集沉淀

### 4. 知识调用系统

这是“教材”的替代物。

当项目需要知识时，系统即时生成学习路径：

- 解释概念
- 推荐材料
- 生成练习
- 做苏格拉底式追问
- 提供多难度版本
- 把知识连接到当前项目

知识不是被取消，而是从固定课程变成可调用资源。

### 5. 同伴协作系统

这是“班级”的替代物。

它按项目、兴趣和成长阶段组队，而不是按年龄固定编班。

核心功能：

- 小队组建
- 角色分工
- 任务看板
- 同伴反馈
- 展示会
- 冲突复盘
- 共同作品

### 6. 作品集与评价系统

这是“考试和文凭”的替代物。

评价证据包括：

- 作品证据
- 过程证据
- 协作证据
- 人格成长证据
- 外部反馈

它要回答的不是“你考了多少分”，而是“你做出了什么，你如何做成，你在成为怎样的人”。

### 7. 家长支持系统

家长不再扮演监工，而是扮演成长环境的维护者。

功能：

- 查看阶段报告
- 理解孩子兴趣
- 获得陪伴建议
- 提供家庭任务
- 记录成人观察
- 设置安全边界
- 管理隐私权限

### 8. 现实反馈市场

这是系统连接真实世界的接口。

反馈来源：

- 家人
- 同伴
- 社区成员
- 真实用户
- 创作者
- 工程师
- 商家
- 公益组织
- 领域专家

孩子的项目必须尽量得到真实反馈，而不是只在系统内部完成。

### 9. 安全与监护系统

这是儿童产品的底层系统。

必须包含：

- 年龄分级
- 内容安全
- 陌生人接触限制
- 家长授权
- 数据最小化
- 敏感话题升级
- 危机识别
- 人类监护介入
- 可解释的推荐记录

## 2. 硬件落地形态

### 第一阶段：轻硬件，先用现有设备

适合 90 天试点。

设备组合：

- 平板电脑：项目面板、视频、画图、阅读、作品整理
- 手机：拍照、录音、外出任务、即时记录
- 蓝牙耳机：语音陪伴和户外任务
- 家庭电脑：长文写作、编程、作品编辑
- 打印机或手账本：把数字任务落到纸面

这一阶段不要先做自研硬件。先验证教育机制是否成立。

### 第二阶段：家庭成长终端

当试点稳定后，可以设计一个家庭固定设备。

形态：

- 桌面小屏
- 麦克风
- 摄像头
- 扬声器
- 实体静音键
- 家长控制键
- 本地加密存储

用途：

- 早晚对话
- 每日计划
- 项目提醒
- 家庭共创
- 睡前复盘
- 作品展示

它不应该像监控摄像头，而应该像家庭中的一盏成长灯。

### 第三阶段：儿童随身伙伴

适合更成熟的版本。

形态可以是：

- 儿童手表
- 语音徽章
- 夹在衣服上的小设备
- 骨传导耳机
- 低干扰的随身录音按钮

主要用途：

- 户外观察记录
- 语音提问
- 任务提醒
- 安全定位
- 紧急联系
- 项目证据采集

注意：随身硬件必须低打扰，不能把孩子变成被实时监控的对象。

### 第四阶段：项目工具包

这是最重要但最容易被忽视的硬件。

教育不能只靠屏幕。

项目工具包包括：

- 测量工具
- 观察放大镜
- 简单传感器
- 积木和结构件
- 纸笔和卡片
- 手工材料
- 基础电子套件
- 植物种植套装
- 采访卡
- 展示板

AI 负责引导，工具包负责把孩子带回真实世界。

## 3. 大模型选型

模型不能只选一个。教育系统需要模型路由。

### 模型路由原则

| 场景 | 需要的能力 | 推荐模型层 |
| --- | --- | --- |
| 日常陪伴 | 低延迟、稳定、温和、多轮记忆 | 轻量高性价比模型 |
| 项目规划 | 推理、拆解、长期上下文 | 强推理模型 |
| 作品反馈 | 写作、审美、结构化评价 | 高质量语言模型 |
| 语音对话 | 实时音频、低延迟、多模态 | Realtime / Live 模型 |
| 图像观察 | 视觉理解、场景识别 | 多模态模型 |
| 家长报告 | 长上下文、总结、风险识别 | 强推理 + 安全模型 |
| 本地隐私 | 离线、低成本、基础理解 | 小型本地模型 |

### 推荐的 1.0 组合

截至 2026-07-04，建议采用多供应商路由，而不是单模型绑定。

#### 主力云端模型

OpenAI：

- `gpt-5.5`：用于复杂项目规划、深度复盘、家长报告和高质量作品反馈。
- `gpt-5.4-mini`：用于日常对话、普通知识解释、低成本高频任务。

理由：OpenAI 官方模型文档将 `gpt-5.5` 定位为复杂推理和编码的旗舰模型，并建议低延迟和低成本任务使用 `gpt-5.4-mini` 或 `gpt-5.4-nano`。最新 OpenAI 模型支持文本和图像输入、文本输出、多语言和视觉能力。

Anthropic：

- `claude-sonnet-5`：用于长对话、写作反馈、人格化陪伴、项目导师风格对话。
- `claude-haiku-4.5`：用于低成本快速反馈和简单分类。

理由：Anthropic 官方模型文档将 `claude-sonnet-5` 描述为速度和智能的最佳组合，将 `claude-haiku-4.5` 描述为接近前沿智能的最快模型。

Google Gemini：

- `gemini-2.5-flash`：用于高频低延迟任务。
- `gemini-2.5-pro`：用于复杂任务、深度推理和代码能力。
- `gemini-3.1-flash-live-preview` 或同类 Live 模型：用于实时语音和视频对话。

理由：Google 官方 Gemini 文档将 `gemini-2.5-flash` 定位为低延迟高容量任务的价格性能模型，将 `gemini-2.5-pro` 定位为复杂任务的高级模型，并提供面向实时语音对话的 Live 模型。

### 本地模型层

本地模型不负责高质量教学决策，而负责低风险、隐私相关和离线任务：

- 唤醒词识别
- 简单意图分类
- 本地笔记摘要
- 家庭设备控制
- 离线任务提醒
- 敏感内容预筛查

可选择轻量开源模型，但 1.0 阶段不把教育质量押在本地模型上。

## 4. 推荐技术架构

### 前端

- 儿童端 App：平板优先，手机兼容
- 家长端 App：手机优先
- 家庭终端：Web / 原生壳
- 项目展示页：Web

### 后端

- 用户和权限系统
- 成长档案服务
- 项目任务服务
- 作品集服务
- 模型路由服务
- 内容安全服务
- 通知和节奏服务
- 审计日志服务

### 数据层

- 结构化数据库：用户、项目、任务、权限
- 向量数据库：成长记忆、项目资料、作品索引
- 对象存储：图片、音频、视频、作品文件
- 本地缓存：家庭终端和儿童设备的离线记录

### AI 编排层

核心不是直接调模型，而是建立 Agent 工作流：

- 成长伙伴 Agent
- 项目导师 Agent
- 知识讲解 Agent
- 复盘 Agent
- 安全监护 Agent
- 家长报告 Agent
- 作品评价 Agent

不同 Agent 使用不同模型，并通过统一的成长档案读写。

## 5. 最小可行产品

第一版不要做全系统。

MVP 只做五件事：

1. 儿童端 AI 成长伙伴
2. 项目任务面板
3. 每日语音或文字复盘
4. 作品集记录
5. 家长周报

硬件先用：

- 平板
- 手机
- 蓝牙耳机
- 纸质项目卡

模型先用：

- 一个主力高质量模型
- 一个低成本日常模型
- 一个实时语音模型
- 一个内容安全和监护流程

## 6. 首个产品场景

最适合的首个场景不是“学数学”，而是“30 天好奇心项目”。

孩子每天做三件事：

1. 提出或选择一个问题
2. 做一次真实观察、采访、制作或验证
3. 晚上和 AI 成长伙伴复盘并沉淀到作品集

30 天后，每个孩子必须完成：

- 30 条问题记录
- 10 次真实观察
- 3 次外部反馈
- 1 个可展示作品
- 1 份成长报告

这个场景能最小成本验证：孩子是否愿意持续使用，AI 是否真的能把孩子带向真实世界，家长是否看得见成长。

## 7. 关键判断

这个系统真正的落地点不是模型，也不是硬件，而是闭环：

孩子提出问题，AI 帮他拆成项目，硬件帮他进入真实世界，项目产生作品，真实世界给反馈，反馈进入成长档案，成长档案再反过来塑造下一次探索。

如果这个闭环跑通，教育就不再依赖学校和老师作为中心。
